What I think this application does:
It seems to be a task management app — basically a more advanced to-do list. Looking at the class names (Task, TaskPriority, TaskStatus, TaskManager),
I'm guessing users can create tasks, give them a priority level, set a due date, and mark them as done. There's also something about tags and statistics, 
so maybe it tracks how many tasks you've completed too.

Technologies/frameworks I think it uses:
I don't see any annotations like @Service or @Controller, so I don't think this uses Spring. It looks like plain Java to me.
I noticed java.time.LocalDateTime and java.time.LocalDate being used, so it's probably Java 8 or newer. I also see .stream() being used in a few places,
which is also a Java 8+ feature. I'm not sure if there are any external libraries — I didn't see any in TaskManager.java, but I haven't checked the build file yet.

Folder structure pattern I observed:
Based on the package names (taskmanager.app, taskmanager.model, taskmanager.storage), it looks like the code is split by responsibility rather than by feature.
So all the data classes are grouped together in model, anything related to saving/loading data is in storage, and the main logic that ties everything together 
is in app. This feels like a fairly standard layered structure.

Things I was confused about:
I couldn't find a main() method anywhere, so I don't actually know how you're supposed to run this program.
In updateTaskStatus and updateTaskPriority, there's this odd pattern where it creates a brand new Task object just called "tempTitle" and only sets one field on it
before passing it to storage.updateTask(). That felt strange to me — why create a whole new object instead of just updating the existing one directly? I assumed maybe
TaskStorage was somehow smart enough to merge only the changed fields, but I wasn't sure.
I also wasn't sure where the actual data gets stored — a database? A file? Nothing in TaskManager.java made that obvious.
