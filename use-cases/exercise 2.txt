My initial search
I searched the codebase for terms like "csv", "export", "write", and "file". I didn't find anything called "export" or "CSV" anywhere — this feature 
doesn't exist yet. But I did find something related: TaskStorage.java already has file-writing logic in its save() method, which uses FileWriter and a 
Gson object to write all tasks out to a JSON file. So there's no export function, but there is an existing pattern for writing task data to a file.
I also noticed TaskStorage has a getAllTasks() method that returns every task as a List<Task> — that seems like it would be useful for export too, 
since I'd need to loop over every task to write it to a CSV.

My hypothesis
Based on what I found, I think a CSV export feature would most likely belong in TaskStorage, right next to the existing save() method, since that's 
already where file-writing happens. I'm guessing I'd add something like an exportToCsv(String filePath) method there.
I think TaskManager would also need a small change — probably a new public method like exportTasks(String filePath) that just calls the new TaskStorage 
method, the same way createTask() calls storage.addTask(). That seems to be the pattern this codebase follows — TaskManager doesn't do the actual work 
itself, it delegates to TaskStorage.
